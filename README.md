# CircuitGen / GenEE /DeepGen / ElecGenius/ ElecGenieus

一款自动化电路工程生成器，旨在降低电路设计的门槛，提高专业人士的设计效率。用户输入一段话或图片，系统生成一个完整的电路工程项目（电路原理图，连接教程，代码）。

## 项目方案

### 理解需求

系统接收用户输入的文本描述或图片，理解用户对电路功能的需求，并进行需求文档提示词工程，生成结构化的需求文档。

可选：图片输出功能描述和优化建议（成本和性能）。

#### 需求文档提示词工程

- 项目名称+描述:  用户可以输入项目的简要描述，例如 "用按钮控制 LED 灯闪烁"。
- 使用的主控:  默认使用 Arduino，可以由用户显式指定 (例如 ESP32)，也可以由系统推荐。
- 建议使用的传感器:  识别和推荐合适的传感器。
- 元器件预算:  用户可以输入预算范围，系统可以根据预算智能推荐合适的元器件。
- 代码功能:  用户可以输入代码的功能要求，例如 "按钮控制 LED 灯的闪烁"，串口打印"Hello, World!"。

输出电路工程需求文档，以下为模板：

```markdown
项目名称: 用按钮控制LED灯闪烁
项目描述: 用户可以通过按下按钮，控制 LED 灯的闪烁。
主控类型: Arduino Uno
传感器: 按钮, LED
元器件预算: 50 元
代码要求: Arduino C/C++
代码功能: 按钮控制 LED 灯的闪烁
```

### 选择元器件

元器件选择: 根据需求文档，从元器件数据库中筛选和选择合适的元器件，并输出详细的元器件列表 (包含购买链接和成本估算，可选)。

元器件RAG: 需要建立一个全面的元器件数据库，包含各种常用电子元器件的详细信息，例如参数、引脚定义、datasheet 等。
兼容性检查: 系统可以进行简单的兼容性检查，例如检查选定的主控是否支持用户选择的传感器。

- 输出详细并且格式处理完成的元器件列表

可选：元器件购买链接+成本估算（参考CompleteBOM）
可选：一个交互界面，方便用户进行元器件选择和管理。

### 设计电路 绘图

电路设计与表格输出：基于选定的元器件和需求，系统自动进行电路设计，并将电路连接信息和元器件参数以表格文件的形式输出。
包含字段:

```markdown
Type (类型): 描述表格行的类型，例如 "Component" (元器件) 或 "Net" (网络)。
Designator (位号): 元器件的唯一标识符，例如 R1, LED1, U1 等。 网络可以自定义名称，例如 VCC, GND, LED_Net 等。
Part (器件类型): 元器件的通用类型，例如 RES (电阻), LED, BUTTON, ARDUINO_UNO 等。
Value (参数值): 元器件的参数值，例如 10k (电阻值), Green (LED 颜色) 等。
Footprint (封装): 元器件的封装形式，用于 PCB 设计 (可选，如果需要后续 EDA 工具处理)。
Description (描述): 元器件或网络的描述信息，方便理解电路功能。
Pin1, Pin2, ..., Pin10 (引脚): 元器件的引脚名称或编号。这里预留了 10 个引脚，你可以根据实际情况调整引脚列的数量。
Net1, Net2, ..., Net10 (网络): 与元器件引脚相连的网络名称。 如果引脚未连接，可以留空。
```

这个表格文件将作为电路原理图的中间表示，详见[circuit_table.csv](circuit_table.csv)

电路图绘制: 使用 schemdraw 库，读取上一步生成的表格文件，将其中的元器件和连接信息转换为图形化的电路原理图,详见[draw_circuit.py](draw_circuit.py)

### 生成代码

根据电路设计和用户需求，生成可直接烧录到主控的代码文件。

### 生成教程

根据电路原理图、元器件列表和代码，自动生成步骤化的电路连接教程和使用说明。

步骤化:
元器件清单复核
模块/电路布局与解释 (可选)
模块连接步骤 (图文结合) ,
代码烧录教程
程序运行和测试
常见问题解答 (FAQ)

可选：视频或者图片教程（使用diagrams生成），支持readthedoc等快速文档部署框架。

## 阶段性目标

### blink 单开发板无需元器件

专注于 代码生成

无需实际元器件，即可验证代码的正确性。
可以先实现一个简单的网页或桌面应用，用户输入 "blink LED"，系统就能生成 Arduino 的 blink 代码，最好并提供对应教程。  

### 充电宝 （无需主控）

在这个阶段，表格文件可以用来描述充电宝的电路，包含电源管理芯片、电池、保护电路等元器件。

需要完善元器件数据库，并开发电路设计算法，自动生成充电宝的电路表格文件，然后使用 schemdraw 绘制电路图。

### 按钮让电机转起来

开始加入 交互功能 和 更复杂的元器件 (电机驱动)。
用户输入 "用按钮控制电机转动"，系统需要能够理解 "按钮控制" 输入和 "电机转动" 输出的意图，需要扩展表格文件模板，支持更多类型的元器件和连接方式，并完善电路设计算法和绘图代码。

添加案例:  在基础功能完善后，可以逐步添加更多更丰富的案例，例如温湿度监控、智能家居控制、简单机器人等，不断扩展系统的应用场景和功能。

### 3D打印机 代码可以打印埃菲尔铁塔代码

涉及 复杂系统集成 和 运动控制

3D 打印机是一个多轴运动控制系统，需要驱动步进电机、控制加热、进行G代码解析等。
实现这个目标，将极大地提升项目的技术深度和应用价值。

### 机器人

最终目标，进一步提升项目的复杂度和实用性。

## 得分要点 (待补充)

多用 NV 生态:  在项目中尽可能多地使用 NVIDIA 的技术和产品。

场景要足够全面: 演示覆盖以下场景：教育场景，DIY 爱好者，工程师辅助设计，快速原型验证

RAG:电路工程案例+教程书籍检索+Datasheet 自动参考（联网自动搜索元器件的 datasheet, 从中提取关键参数和引脚信息，辅助电路设计和代码生成。）

多方案同步生成：系统能同时生成 多种不同的电路设计  或 多种元器件组合方案，并比较各种方案的优缺点、成本、性能等，供用户选择。
